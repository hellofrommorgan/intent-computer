Skill Graph

Capabilities your agent learns and reuses. Not hardcoded tools — composable instructions that grow with how you work.

Skills are instructions, not code

A skill is a set of instructions your agent follows when you ask for it. You type a command, and your agent knows exactly what to do. No setup, no configuration — it just works.

openclaw

Your agent reads the instructions and follows them. The LLM is the runtime — you don't write code for a new capability, you write instructions.

They adapt to your setup

Skills use placeholders that get filled in based on your specific configuration. One person calls their notes "thoughts," another calls them "claims," another calls them "observations" — the same skill works for all of them. You don't fork skills to match your naming conventions.

Built-in skills

The Intent Computer ships with skills in two categories:

Knowledge processing — powered by Ars Contexta:

  • /reduce — extract insights from source material
  • /reflect — find connections, add links, update maps
  • /reweave — reconnect notes that have drifted apart
  • /verify — check quality and consistency
  • /pipeline — run the full processing chain end to end
  • /learn — research a topic using web search, file what you find
  • /seed — add raw material to the processing queue

System management:

  • /next — recommend the most valuable thing to do next
  • /stats — how healthy is your knowledge graph?
  • /graph — ask questions about your graph's structure
  • /remember — capture something you want the system to learn
  • /rethink — review what you've observed and resolve contradictions
  • /tasks — manage the processing queue
  • /health — run diagnostics

Skills chain together

Skills compose into workflows. /pipeline chains /reduce → /reflect → /verify in sequence, with quality checks between each step. /learn chains web search → /seed → /pipeline. Each skill suggests a natural next step, so work flows without you managing it.

You control how autonomous this is:

  • Manual — skills suggest the next step, you decide when
  • Suggested — skills suggest + add to the task queue
  • Automatic — one skill finishes, the next one starts

Writing your own

Create a markdown file with a name, description, triggers, and instructions. The system finds it automatically. A skill can be as simple as "when I say /standup, summarize what changed since yesterday" or as involved as a multi-step research workflow with web scraping, extraction, and connection-finding.

The queue

Skills that process multiple items use a persistent task queue. Tasks move through phases, track attempts, and automatically queue repair tasks with full context when something fails. The heartbeat can pick up queued tasks between sessions — your agent keeps working even when you're not around.
